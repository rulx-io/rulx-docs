## Prerequisites

- Node `20.19.0`
- `pnpm` `10.27.0`
- Go `1.22`
- Docker Compose

The monorepo root is the source of truth for the toolchain and common commands.

## 1. Install dependencies

From the monorepo root:

```bash
pnpm install
```

If you need the pinned `pnpm` version globally:

```bash
npm install -g pnpm@10.27.0
```

## 2. Prepare local configuration

Copy the example environment file and adjust values if needed:

```bash
cp .env.example .env.local
```

The local stack defaults to:

- controller on `http://localhost:3000`
- gateway on `http://localhost:4000`
- workspace binding `ws_integration` for HTTP trigger testing

## 3. Start the main local stack

```bash
pnpm dev
```

That bootstraps the main local services:

- controller
- web
- node worker
- runner
- gateway

## 4. Create a workspace

Open the web app and create a workspace at `/setup/workspace` if the controller does not already have one.

Workspaces are the primary isolation boundary. They scope:

- URLs
- flows
- schemas
- code modules
- runs
- settings and access

Read more in [Workspaces](/concepts/workspaces).

## 5. Build your first artifact

Inside a workspace, open **Build** and start with one of these:

- a new flow in `/build/flows`
- a new schema in `/build/schemas`
- a new code module in `/build/nodes`

For the typical first pass, create a flow and use the flow editor plus config views to define a callable interface and publish a version.

Read more in [Flows](/build/flows).

## 6. Run and inspect

Use the workspace run surfaces to inspect:

- current and recent runs
- state transitions
- logs
- artifacts produced by a run

The run model is covered in [Runs, timeline, and logs](/concepts/runs-and-logs).

## Optional: use `rulxctl`

`rulxctl` is the controller-focused CLI for developers, operators, and CI:

```bash
rulxctl workspace ls
rulxctl --workspace ws_1234 flow ls
rulxctl --workspace ws_1234 job ls
```

See [rulxctl](/self-host/rulxctl).
