---
title: 'Schemas'
description: 'Workspace-scoped schema management and versioning'
---

Schemas provide explicit, versioned payload contracts for Rulx build surfaces.

## Why schemas matter

With schemas, you can:

- keep inputs and outputs explicit
- pin published versions
- reuse the same contract across multiple artifacts

## Schema workflow

1. create the schema
2. define and revise it in the editor
3. publish a version
4. bind other artifacts to the published version
