---
title: 'Code modules'
description: 'Custom nodes managed as workspace artifacts'
---

Code modules are the custom node surface in Rulx.

They let you package reusable behavior as workspace-managed artifacts that flows can depend on.

## What makes them different

Compared with flows and schemas, code modules have a stronger dependency-management story.

That shows up in:

- module registry pages
- editor-generated source
- node dependency installation jobs
- workspace package registry configuration
