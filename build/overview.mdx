title: 'Build overview'
description: 'How the Rulx build surface is organized'
---

The **Build** area in the web app is the home for authoring workspace artifacts.

It is organized around three artifact families:

- flows
- schemas
- code modules

## Flows

Flows are the orchestration artifacts your team builds and runs.

## Schemas

Schemas keep payload contracts explicit and versioned.

## Code modules

Code modules are the custom node surface for reusable behavior.

## Publishing

Although each artifact type has its own authoring flow, drafts and published versions are shared ideas across the build surface.
