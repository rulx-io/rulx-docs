The runs surface is where authoring turns into runtime debugging.

## Typical tasks

Use the runs pages to:

- list recent execution history
- open a specific run
- inspect logs and timeline
- review artifacts produced by a run
- cancel work that should stop

## Workspace-wide vs flow-specific

Rulx gives you two useful views:

- workspace-wide runs for broad operational awareness
- per-flow runs when you are focused on one artifact

The data model behind both surfaces is the same idea covered in [Runs, timeline, and logs](/concepts/runs-and-logs).
